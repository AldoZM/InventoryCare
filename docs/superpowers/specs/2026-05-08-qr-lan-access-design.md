# QR LAN Access — Design Spec
Date: 2026-05-08
Status: Approved

## Goal

Allow workers and admin to connect their phones to InventaryCare by scanning a QR code in the Settings section, without needing to type a URL manually.

## Context

The app runs on HTTPS (`https://192.168.x.x:8080`) so camera access works on LAN phones. The problem: non-technical users don't know what URL to type. A QR code in Settings solves this — admin opens Settings, shows the QR to a worker, worker scans with phone camera.

## Approach

- Backend: new endpoint detects LAN IP at request time (socket trick)
- Frontend: lazy-loads a bundled QR library, renders QR in a new settings card
- Library: `qrcode.min.js` (davidshimjs/qrcodejs, ~10KB) bundled locally for offline use

## Architecture

### Backend: `app/routers/system.py`

New file. One endpoint:

**`GET /api/system/lan-url`** — no auth required
- Opens UDP socket toward `8.8.8.8:80`, reads `getsockname()[0]`
- Returns `{ "url": "https://192.168.1.42:8080", "lan_ip": "192.168.1.42" }`
- If detection fails: returns `{ "url": null, "lan_ip": null }`
- Port hardcoded to `8080` (matches `PORT` constant in launcher)
- Registered in `app/main.py` with prefix `/api/system`

### Frontend: `www/js/qrcode.min.js`

Downloaded from `https://raw.githubusercontent.com/davidshimjs/qrcodejs/master/qrcode.min.js` and saved locally. Loaded lazily via `<script>` tag injection inside `renderSettings()`, only when the settings view renders.

### Frontend: `www/js/views/settings.js`

New card added at the top of the settings grid (before backup/restore cards):

```
┌─────────────────────────────────────┐
│ 📱 Acceso desde teléfono            │
│                                     │
│ Escanea este código con la cámara   │
│ de tu celular para abrir            │
│ InventaryCare en tu teléfono.       │
│                                     │
│  ┌──────────┐                       │
│  │  [QR]    │                       │
│  └──────────┘                       │
│  https://192.168.1.42:8080          │
└─────────────────────────────────────┘
```

**State: IP detected**
- QR code (150×150px) generated by `QRCode` lib pointing to `https://{lan_ip}:8080`
- URL shown as text below QR for reference

**State: No IP / no network**
- Message: "No conectado a red local. Conecta la computadora a una red WiFi."
- No QR shown

**Loading sequence:**
1. `renderSettings()` injects `<script src="/js/qrcode.min.js">` if not already loaded
2. Fetches `/api/system/lan-url`
3. On load: renders appropriate state

## Data Flow

```
Admin opens Settings
  → renderSettings() injects qrcode.min.js (once)
  → fetch GET /api/system/lan-url
      → backend detects LAN IP via socket
      → returns { url, lan_ip }
  → if url: generate QR + show URL text
  → if !url: show "no network" message
```

## Error Handling

- Fetch fails (server down): show "Error al obtener dirección de red"
- `lan_ip` is null: show "No conectado a red local"
- QRCode lib fails to load: show URL as plain text only (fallback)

## Files Changed

| File | Action |
|------|--------|
| `app/routers/system.py` | Create — LAN URL endpoint |
| `app/main.py` | Modify — register system router |
| `www/js/qrcode.min.js` | Create — bundled QR library |
| `www/js/views/settings.js` | Modify — add QR card |

## Not In Scope

- QR on dashboard or other views
- Worker-accessible QR (Settings is admin-only by design)
- Refresh button (URL only changes if IP changes, which requires app restart anyway)
